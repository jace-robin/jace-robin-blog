What is Gitpod?

Gitpod is an oline service that lets you access code on github through a github account, and has every feature in Visual Studio Code, a popular IDE made by Microsoft.

It has thousands of extensions that help you code faster, and more effectively. I use many extensions that greatly speed up the tedious parts of coding. They will be listed further down in the tutorial, and I highly advise them. You'll thank me later, trust me

Signing in to Gitpod

To start, visit gitpod (again, click to be redirected)

Next, click sign in with github. If you are already signed in on github, it should automatically sign you in, but if not, just type in the username and password you used prior.

Also,if it asks for permissions now, check every box. This is extremely important, do not miss a box!

Adding a Repository

For this next step, you will need to open the repository you made previously on github.

The process for adding a new repo to your gitpod workspaces is simple luckily.

First, copy the URL in the search bar. You need to have 'https://' before though, so after clicking, just click once more a bit after the first, this will expand it so that you can copy the entire URL.

Next, make a new browser tab. Type in 'https://gitpod.io/#' and then paste the url you copied. For example: https://gitpod.io/#https://github.com/john-doe/john-doe.github.io

Gitpod will automatically create a workspace for you with the files in the repo

Adding Permissions

To be able to update the code on github, you will need to give Gitpod permissions to use your account

To do so, make a new browser tab. Visit gitpod.io again, and click your profile once it loads. This can be found on the top right of the page, and has a profile picture provided by your github account. Alternatively, you can just click Settings on the main page

Click 'Integrations'. Here, you can see account associated with your gitpod account. Click the three vertical dots to the right of the github account, then 'Edit Permissions'

Next, make sure every box is clicked. Once done, click 'Update Permissions'

Using Your New Workspace

Your code can now be accessed by clicking 'Workspaces' on the home page of gitpod. It is also important that you click the three dots next to it, and click 'Pin Workspace'. If you do not, then after 14 days of inactivity, teh gitpod workspace will be deleted, and any changes not sent to github will be lost

To open the workspace, click the name, and you will be redirected to the IDE

In terms of use, it should work fine for now, but some extensions will help you greatly

Extensions

Extensions provide many useful tools that are vital for coding quickly

To add them, click the icon that looks like 3 stacked boxes, with one lifted.

There are a few extensions installed automatically by gitpod, but heres a few I use a lot, and how to use them effectively

Beautify-HookyQR(auto indent code for better readability)

Github Pull Requests And Issues-Github(used to pull updates to your github repo, if updated from an external source

Live Server-ritwickdey(allows you to open a test page containing the content of your current file)


I recommend binding Beautify File(A beautify command) to a key combination. I use 'ctrl+shift+a', but whatever fits you better will work just as well.

To do this, hold 'ctrl+shift+p' and type in Beautify. One of the first results should be Beautify File. Click the gear icon, then +. hold whichever key combination you want, and then press enter. It may be as short or as long as you like, just remember that you might use this a lot!


You may also not like how your workspace looks, which is fine, because white light can be blinding.

Luckily, you can add extensions that change the colors and syntax highlighting of your code! Simply look up themes in the search bar, and look around for one you like.

My current theme is Raspberry Candy(original). It all comes to your preference though, there are tons of nice themes, and they are fun to look through and find one you enjoy using!

Git Commands

Git commands are bits of code that are typed into the terminal at the bottom of your screen. If you dont see it, dont worry, you can make a new one by pressing the three lines on the top right of your screen, and then hovering over terminal. Clicking 'New Terminal' should make a new one appear!

There are three main commands that you will use for now. These should be typed in order every time you want to send the code to github to update your files


git add .

git commit -m "add a description here of what you changed!"

git push


With these commands, it should push your changes to github. If you visit the repo there, you should see any changes you have made

If you dont understand what each command does, for now, dont sweat it. If you want more information, you can find tons of tutorials online

Another useful command is 'git pull' and 'git merge'

These commands will pull the changes from github that are not on your current device. If you are using multiple computers, this is important to know. Just make sure that you push your current changes with the above commands first, or you will get an error!
